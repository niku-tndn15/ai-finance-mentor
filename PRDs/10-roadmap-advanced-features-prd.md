# UrPaisa Roadmap and Advanced Features PRD

## 1. Objective

Define how UrPaisa can expand after MVP validation while protecting the core mentor-first product direction.

## 2. Phase 1: MVP

Goal:

- Prove users want an AI money coach that gives actions.

Included:

- Quick onboarding.
- Income and expense input.
- Safe-to-use money calculation.
- Goal setup.
- Simple debt input.
- Daily money nudge.
- Done, Skip, Remind me later.
- Weekly reflection.
- Basic learning.
- Conversational money questions.

## 3. Phase 2: Habit Depth and Persistence

Potential features:

- User accounts.
- Persistent profile.
- More goals.
- Better debt tracking.
- Formal habit challenges.
- Adaptive tone.
- Money Health Score.
- Monthly reset.
- Notification timing optimization.

## 4. Phase 3: Automation and Data Enrichment

Potential features:

- Bank account integration.
- Automatic transaction categorization.
- Subscription detection.
- Salary detection.
- Credit card due date tracking.
- Bill reminders.
- Actual versus planned spend comparison.

Constraints:

- Integrations require explicit consent.
- Manual mode must remain available.
- Security and regulatory review required before launch.

## 5. Phase 4: Advanced Mentorship

Potential features:

- AI Financial Twin simulations.
- Salary day autopilot.
- Emergency mode.
- Purchase decision assistant.
- Life event mode.
- Family finance mode.
- Investment readiness guidance.
- Risk alerts.

## 6. Advanced Feature Definitions

### Salary Day Autopilot

Creates a salary allocation plan:

- Bills bucket.
- Rent bucket.
- Savings bucket.
- Debt bucket.
- Goal bucket.
- Safe spending bucket.

Example:

"Your salary came in. Here is the safest split: INR 18,000 for rent and bills, INR 5,000 for emergency fund, INR 3,000 for debt, INR 2,000 for travel goal, and INR 12,000 safe to use."

### Life Event Mode

User can tell UrPaisa about:

- New job.
- Moving cities.
- Marriage.
- Higher rent.
- Education loan repayment.
- Medical expense.
- Travel plan.
- Family support increase.
- Salary delay.
- Job loss.

UrPaisa adjusts the plan.

### Emergency Mode

Triggered by:

- Low balance.
- Income delay.
- Job loss.
- Urgent expense.

Priorities:

- Essential bills.
- Rent.
- Food.
- Debt minimum payments.
- Medical needs.
- Avoiding penalties.
- Temporary spending freeze.

Example:

"You are in a tight cash period. For the next 10 days, focus only on essentials. You can safely spend INR 280 per day."

## 7. Roadmap Guardrails

Future features should be added only if they:

- Improve next-best-action quality.
- Reduce user effort.
- Improve trust.
- Help build better money habits.
- Make financial tradeoffs easier to understand.

Delay features if they:

- Turn UrPaisa into a passive dashboard.
- Require sensitive data before trust is earned.
- Create product-selling pressure.
- Add complexity without improving action.
