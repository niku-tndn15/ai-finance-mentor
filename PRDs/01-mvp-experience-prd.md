# UrPaisa MVP Experience PRD

## 1. Objective

Build the first version of UrPaisa as an AI money mentor that gives users one clear financial action at a time.

The MVP must prove this product hypothesis:

Users will engage more with personalized next-best money actions than with passive charts or manual budgeting dashboards.

## 2. First-Time User Journey

1. User opens UrPaisa.
2. App explains that it is a money mentor, not a boring tracker.
3. User enters income, payday, fixed expenses, debts, and goals.
4. UrPaisa calculates safe-to-use money.
5. UrPaisa identifies the user's first financial risk.
6. User receives one simple action.
7. User taps Done, Skip, or Remind me later.
8. UrPaisa learns from that response.
9. User gets a weekly reflection.
10. UrPaisa adjusts the next week's money plan.

## 3. Daily Journey

1. User opens app.
2. UrPaisa shows safe-to-use money.
3. User sees one recommended action.
4. User can ask a money question.
5. User completes or skips the action.
6. Product updates the financial model.

## 4. Weekly Journey

1. User receives weekly reflection.
2. UrPaisa shows progress and patterns.
3. User gets one financial habit to focus on.
4. User can choose a challenge for the next week.

## 5. Monthly Journey

1. User gets monthly reset.
2. UrPaisa compares planned versus actual behavior using available user-entered and interaction data.
3. User updates income, expenses, debts, and goals.
4. UrPaisa creates a new monthly money plan.

Monthly reset can be lightweight in MVP and richer after transaction integrations exist.

## 6. MVP Screens

### Welcome Screen

Purpose:

- Introduce UrPaisa as an AI money mentor.

Required content:

- App name: UrPaisa.
- Promise: "Know what is safe to spend and what money move to make next."
- CTA: Get started.
- Note: "No bank linking required for the first version."

### Onboarding Screen

Purpose:

- Capture user type, income pattern, goal, and coaching preference.

Inputs:

- User type: working professional, student with income, freelancer/gig worker, other.
- Income type: fixed salary, variable income, mixed income.
- Primary goal.
- Coaching tone.

### Financial Setup Screen

Purpose:

- Build the first financial situation model.

Inputs:

- Monthly income.
- Payday.
- Fixed expenses.
- Subscriptions.
- Simple debt input.
- Primary goal.
- Planned savings.

### Safe-to-Use Money Screen

Purpose:

- Show the user's safe spending capacity.

Required output:

- Safe-to-use money this month.
- Safe weekly spend.
- Risk zone explanation.
- One plain-language summary.

### Daily Mentor Screen

Purpose:

- Show the core daily experience.

Required elements:

- Safe-to-use money meter.
- One recommended action.
- Why this action matters.
- Done, Skip, Remind me later.
- Ask UrPaisa input box.

### Weekly Reflection Screen

Purpose:

- Show progress and next focus.

Required elements:

- What went well.
- What did not go well.
- Pattern detected.
- One recommended action.
- Optional challenge.

## 7. Example MVP Experience

User profile:

- Income: INR 60,000.
- Rent: INR 18,000.
- EMIs: INR 8,000.
- Bills: INR 5,000.
- Planned savings: INR 7,000.
- Goal: Emergency fund.
- Debt: Credit card balance INR 20,000.

UrPaisa calculates:

- Fixed expenses: INR 31,000.
- Planned savings: INR 7,000.
- Debt priority buffer: INR 5,000.
- Safe-to-use money: INR 17,000.

App response:

"You have INR 17,000 safe to use this month. Your top priority is credit card debt because it has high interest. Your best action today is to pay INR 2,000 extra toward your credit card before weekend spending starts."

User taps Done.

UrPaisa learns:

- User responds to debt payoff actions.
- User is active after payday.
- User completes small clear actions.
- Future nudges should focus on debt and saving early.

## 8. MVP Acceptance Criteria

- User can complete onboarding and financial setup in under 5 minutes.
- User can set at least one goal and one debt item.
- App calculates safe-to-use money.
- App shows one recommended daily action.
- User can ask at least basic money questions.
- Done, Skip, and Remind me later are stored.
- Weekly reflection uses actual user responses.
- The experience does not depend on bank linking.
